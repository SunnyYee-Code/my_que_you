## 3. 关键里程碑

| 里程碑                  | 目标                                                  | 交付物                          | 风险检查点                                         |
| ----------------------- | ----------------------------------------------------- | ------------------------------- | -------------------------------------------------- |
| M1：V1 / V2 边界冻结    | 明确 V1 已交付与 V2 必做范围                          | 路线图、V2 需求文档、后续计划   | 是否仍有 V2 / V3 范围混淆                          |
| M2：V2 需求冻结         | 明确 V2 P0 / P1 拆解、依赖与验收口径                  | V2 任务包、PRD、验收标准        | 合规、风控、消息链路是否有缺失                     |
| M3：V2 核心能力开发提测 | 合规、安全、成局效率、增长基础能力完成联调            | 可提测版本、测试清单            | 注销、实名、拉黑、提醒、邀请链路是否闭环           |
| M4：V2 上线复盘         | 对核心城市灰度放量验证                                | 数据复盘、问题清单              | 成局率、到场率、举报率、裂变转化是否改善           |
| M5：V3 规划启动         | 基于 V2 数据决定收尾顺序、运营深化与社区 / 商业化投入 | V3 需求文档、资源评估、收尾清单 | V1/V2 遗留闭环是否明确；留存与增长是否支撑 V3 投入 |

## 4. 风险提醒

- 若账号注销、实名认证等合规链路未先补齐，不建议放大投放和用户增长。
- 若黑名单、评价档案等信任能力缺失，平台扩大后纠纷和举报风险会快速上升。
- 若开局提醒、紧急补位、推荐排序落地效果不佳，V2 对成局率和到场率的改善会受限。
- 若邀请码、分享海报缺少归因与转化监测，增长能力上线后难以判断真实投入产出。
- 若 V1 / V2 遗留项不在 V3 开头统一收口，会造成“文档已完成、真实能力未完成”的错配。
- 若将社区、商业化和扩展能力继续提前压过收尾项，会削弱 V3 的上线价值和执行聚焦。
