# 雀友聚产品路线图

## 1. 路线图目标

围绕“更快成局、更高到场、更低纠纷”三个核心目标，分阶段建设雀友聚的供给、匹配、履约与运营能力。

## 2. 规划原则

- **先闭环后增长**：优先保证用户能完成找局、建局、入局、沟通、履约完整链路。
- **先信任后扩张**：先补齐信用、举报、评价等基础约束，再放大规模。
- **先重点城市后复制**：以核心城市验证模型，再复制到更多城市。
- **版本边界清晰**：统一使用 V1、V2、V3 版本命名；V2 聚焦近期必须落地能力，V3 承接社区、商业化与扩展探索。

| 版本 | 时间建议 | 主题 | 核心交付 | 成功标准 | 主要仓库 |

## 5. 按仓库拆解

### 5.1 my_que_you

**定位：** Web 管理与业务主站能力承载。

#### V2

- 账号注销完整闭环
- 实名认证与认证状态管理
- 黑名单屏蔽与互动隔离
- 紧急补位、推荐排序、开局提醒
- 标签化评价与履约档案增强
- 分享海报、邀请码归因

#### V3

- 常用地点收藏、信用勋章系统
- banner / 活动位、消息触达增强
- 数据看板与性能测试优化
- 雀友圈、俱乐部 / 固定牌友圈、长期局
- 战绩排行榜、记分器、番数计算器
- 会员体系与增值曝光
- 多品类局扩展

### 5.2 que_you_app

**定位：** 移动端深度体验与高频触达承载。

#### V2

- 更轻量的找局 / 入局路径
- 开局提醒、审核结果、群消息推送
- 分享拉新与好友邀请
- 紧急补位消息承接与关键状态提醒

#### V3

- 移动端履约签到 / 到场确认
- 实时位置辅助到店、位置消息发送
- 语音消息能力
- 个人信用档案、实名认证状态、历史局记录沉淀

## 7. 风险提醒

- 若账号注销、实名认证等合规链路未先补齐，不建议放大投放和用户增长。
- 若黑名单、评价档案等信任能力缺失，平台扩大后纠纷和举报风险会快速上升。
- 若开局提醒、紧急补位、推荐排序落地效果不佳，V2 对成局率和到场率的改善会受限。
- 若邀请码、分享海报缺少归因与转化监测，增长能力上线后难以判断真实投入产出。
- 若将社区、商业化和扩展能力继续混入 V2，会造成范围膨胀，影响近期主目标交付。

## 8. 对老板的阶段同步口径

- **当前目标：** 统一版本命名为 V1 / V2 / V3，并重新明确 V2 只承接近期必须实现的能力。
- **当前进展：** 已完成产品优先级文档重排，将需要在 V2 实现的能力收敛到合规、信任、成局效率和增长基础。
- **当前风险：** 若不继续约束 V2 范围，容易再次混入社区、商业化和中长期能力，导致排期失焦。
- **下一步：** 按新的 V2 范围更新需求文档与任务执行计划，再同步研发做拆解评估。
