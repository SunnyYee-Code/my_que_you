# 雀友聚产品路线图

## 1. 路线图目标

围绕“更快成局、更高到场、更低纠纷、更强留存”四个核心目标，分阶段建设雀友聚的供给、匹配、履约、运营与社区能力。

## 2. 规划原则

- **先闭环后增长**：优先保证用户能完成找局、建局、入局、沟通、履约完整链路。
- **先信任后扩张**：先补齐信用、举报、评价等基础约束，再放大规模。
- **先重点城市后复制**：以核心城市验证模型，再复制到更多城市。
- **版本边界清晰**：统一使用 V1、V2、V3 版本命名；V2 聚焦近期必须落地能力，V3 同时承接收尾项、运营深化、社区沉淀与商业化探索。

| 里程碑 | 目标 | 交付物 | 风险检查点 |

## 7. 风险提醒

- 若账号注销、实名认证等合规链路未先补齐，不建议放大投放和用户增长。
- 若黑名单、评价档案等信任能力缺失，平台扩大后纠纷和举报风险会快速上升。
- 若开局提醒、紧急补位、推荐排序落地效果不佳，V2 对成局率和到场率的改善会受限。
- 若邀请码、分享海报缺少归因与转化监测，增长能力上线后难以判断真实投入产出。
- 若 V1 / V2 遗留项不在 V3 开头统一收口，会造成“文档已完成、真实能力未完成”的错配。
- 若将社区、商业化和扩展能力继续提前压过收尾项，会削弱 V3 的上线价值和执行聚焦。

## 8. 对老板的阶段同步口径

- **当前目标：** 在 V2 基础上启动 V3 规划，先统一收口遗留闭环，再推进运营深化、社区沉淀和商业化探索。
- **当前进展：** 已完成 V3 需求范围重整，将 V1 / V2 未完全闭环项与原 V3 规划能力合并管理。
- **当前风险：** 若直接进入社区和商业化建设，而不先补齐真实环境闭环与回归体系，后续放量风险较高。
- **下一步：** 按新的 V3 范围补齐任务执行计划、测试用例与验收文档，并与研发确认 P0 / P1 拆解顺序。
