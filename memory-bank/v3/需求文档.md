# 雀友聚 V3 需求文档

## 1. 文档目标

本文件用于明确 `my_que_you` 在 V3 阶段需要承接的产品需求范围。

V3 需求文档只保留两类内容：

1. V2 中尚未完成、需要在 V3 继续补齐的功能需求；
2. 产品优先级文档中已明确进入 V3 的新功能需求。

本文件只描述产品需求范围；交付、验证和实施相关内容应维护在其他文档中。

---

## 2. V3 阶段目标

V3 的目标是在不扩散范围的前提下，完成以下三件事：

1. 补齐 V2 遗留的核心产品功能；
2. 提升运营效率、履约体验与用户留存；
3. 推进社区沉淀与商业化探索。

---

## 3. V3 版本边界说明

- **V1**：只记录已经实现的核心闭环能力，不在 V3 需求文档中重复展开。
- **V2**：记录 V2 计划实现但尚未完成的功能，这部分由 V3 继续承接。
- **V3**：承接 V2 遗留功能补齐，以及产品优先级中已经明确进入 V3 的新增需求。

V3 的边界原则：

- 不把“尚未验证”当作“尚未实现”写入需求文档；
- 不把交付动作写成产品需求；
- 不把已经在 V1 中实现的能力重新定义为 V3 新需求；
- 所有 V3 需求都必须能对应明确的产品价值。

---

## 4. V3 需求范围

### 4.1 V2 遗留功能补齐

这部分仅承接 `memory-bank/v2/需求文档.md` 中已定义、但需要在 V3 继续完成的功能需求。

本节按 V2 需求范围整体承接，实际完成状态以单独的实现盘点结果为准。

| 需求项                   | 目标说明                                                       | 价值                               |
| ------------------------ | -------------------------------------------------------------- | ---------------------------------- |
| 账号注销完整闭环         | 支持用户发起注销申请、冷静期、账号状态流转、数据处理与结果通知 | 满足合规要求，降低账号生命周期风险 |
| 实名认证                 | 建立实名认证流程、认证状态展示与风控校验                       | 增强平台信任感，降低虚假身份风险   |
| 黑名单屏蔽               | 支持用户拉黑、屏蔽互动与消息隔离                               | 提升用户安全感与社交可控性         |
| 紧急补位                 | 当拼团临时缺人时，支持快速补位与优先曝光                       | 提升成局率，减少临近开局流失       |
| 推荐排序                 | 基于距离、时间、偏好等信息优化列表排序                         | 提升拼团点击率与申请转化           |
| 开局提醒                 | 在开局前对房主和成员进行提醒                                   | 提升到场率，降低爽约               |
| 标签化评价与履约档案增强 | 在基础评价外增加标签化评价、历史履约档案沉淀                   | 提高匹配质量，降低纠纷             |
| 分享海报                 | 支持拼团分享海报生成与传播                                     | 提升裂变传播效率                   |
| 邀请码                   | 支持邀请码机制与邀请归因                                       | 降低拉新成本，支撑渠道归因         |

### 4.2 运营效率与履约体验增强

| 需求项              | 目标说明                                                                 | 价值                           |
| ------------------- | ------------------------------------------------------------------------ | ------------------------------ |
| 常用地点收藏        | 支持房主收藏常用开局地点，创建时快捷复用                                 | 降低发团成本，提高重复开局效率 |
| 消息触达增强        | 对审核结果、开局提醒、补位提醒、举报处理等关键状态增强触达方式与召回策略 | 提高关键动作到达率与回流率     |
| banner / 活动位     | 支持首页或社区页配置活动位、主题活动和冷启动投放                         | 提升运营投放效率与活动承接能力 |
| 数据看板            | 提供注册、申请、成局、到场、举报、分享、邀请归因等指标看板               | 让运营和管理侧可以用数据做决策 |
| 履约签到 / 到场确认 | 增加签到、到场确认与爽约记录沉淀能力                                     | 提升履约判断准确度             |
| 性能优化            | 持续优化核心链路体验与系统稳定性                                         | 支撑城市放量和更高频运营动作   |

### 4.3 社区关系沉淀与用户留存增强

| 需求项           | 目标说明                                                           | 价值                       |
| ---------------- | ------------------------------------------------------------------ | -------------------------- |
| 信用勋章系统     | 基于履约、评价、活跃度沉淀可视化勋章与身份标识                     | 强化优质用户识别与正向激励 |
| 局后社交沉淀增强 | 在 V1 已有好友、私信基础上，进一步沉淀复局关系、常约对象和关系标签 | 提升复玩率与长期留存       |
| 雀友圈           | 提供轻量内容发布、动态互动与约局话题沉淀能力                       | 提升社区活跃度和内容留存   |
| 俱乐部           | 支持稳定圈层管理、成员沉淀和固定组织能力                           | 提升高频用户组织效率       |
| 长期局           | 支持固定时间、固定成员或长期排期的局组织模式                       | 承接高复玩用户需求         |

### 4.4 沟通与线下到场协同增强

| 需求项       | 目标说明                                               | 价值                          |
| ------------ | ------------------------------------------------------ | ----------------------------- |
| 语音消息     | 在群聊和私聊中支持语音消息发送与播放                   | 提升高频沟通效率              |
| 位置消息     | 支持发送实时位置或集合点位置消息                       | 提升到场协同效率              |
| 到场辅助链路 | 在已有导航能力基础上，进一步增加集合协同与到场辅助能力 | 降低“找不到人/找不到店”的摩擦 |

### 4.5 商业化与品类扩展探索

| 需求项              | 目标说明                                     | 价值                   |
| ------------------- | -------------------------------------------- | ---------------------- |
| 战绩榜              | 提供用户战绩、胜率或活跃榜单展示             | 增强活跃和荣誉感       |
| 记分器 / 番数计算器 | 提供局中辅助工具，提升使用频率               | 提升工具属性和使用时长 |
| 会员权益            | 探索会员身份、权益包和权益展示体系           | 为商业化打基础         |
| 增值曝光            | 探索拼团置顶、优先曝光等增值服务             | 验证付费意愿与供需效率 |
| 多类型桌游扩展      | 评估从麻将扩展到其他桌游或牌类活动的适配能力 | 验证品类扩张空间       |

## 5. V3 优先级建议

| 优先级 | 需求项                                                                               | 原因                                                                 |
| ------ | ------------------------------------------------------------------------------------ | -------------------------------------------------------------------- |
| P0     | V2 遗留功能补齐                                                                      | 直接影响合规、安全、成局效率、履约和增长基础，是 V3 的首要承接范围   |
| P1     | 常用地点收藏、消息触达增强、banner / 活动位、数据看板、履约签到 / 到场确认、性能优化 | 这些能力直接提升复玩率、运营效率和履约质量，且对现有主链路价值最直接 |
| P1     | 信用勋章系统、局后社交沉淀增强                                                       | 能承接高质量用户资产沉淀，提升留存和关系复用                         |
| P2     | 雀友圈、俱乐部、长期局                                                               | 属于社区深化能力，价值明确，但需要建立在主链路稳定和基础数据充分之上 |
| P2     | 语音消息、位置消息、到场辅助链路                                                     | 对体验有提升，但优先级低于基础收口和运营效率能力                     |
| P2     | 战绩榜、记分器 / 番数计算器、会员权益、增值曝光、多类型桌游扩展                      | 属于中长期活跃和商业化探索，适合在 V3 后段按资源分阶段验证           |
