# 雀友聚 V2 需求文档

## 1. 文档目标

本文件用于承接 `my_que_you` 在 V1 之后仍未实现、但已经明确存在业务价值的需求，作为 V2 阶段的产品迭代范围管理文档。

---

## 2. V2 阶段目标

V2 目标是在 V1 主业务闭环已跑通的基础上，进一步提升平台的安全性、信任度、开团效率与沟通体验，重点补齐 V1 中未落地但已识别出的关键能力。

---

## 3. V2 待实现需求范围

### 3.1 账号与合规能力补齐

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 账号注销完整闭环 | 支持用户发起注销申请、冷静期、数据处理、账号状态流转与结果通知 | 满足合规要求，降低账号生命周期风险 |
| 实名认证 | 建立实名认证流程、认证状态展示与风控校验 | 增强平台信任感，降低虚假身份风险 |

### 3.2 拼团效率增强

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 紧急补位 | 当拼团临时缺人时，支持快速补位与优先曝光 | 提升成局率，减少临近开局流失 |
| 常用地点收藏 | 支持房主保存常用开局地点，快速复用 | 降低建局操作成本，提高发团效率 |

### 3.3 社交安全与信任增强

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 黑名单屏蔽 | 支持用户拉黑、屏蔽互动与消息隔离 | 提升用户安全感与社交可控性 |
| 信用勋章体系 | 基于评价、履约、活跃等行为授予勋章标识 | 提升优质用户识别效率与平台信任氛围 |

### 3.4 沟通体验增强

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 语音消息 | 在群聊和私信中支持发送语音消息 | 提升即时沟通效率 |
| 位置消息 | 在聊天中支持发送位置消息 | 提升线下集合场景的到场效率 |

---

## 4. V2 优先级建议

| 优先级 | 需求项 | 原因 |
| --- | --- | --- |
| P0 | 账号注销完整闭环 | 影响合规要求与账号管理边界 |
| P1 | 实名认证、黑名单屏蔽、紧急补位 | 影响平台信任、安全与成局效率 |
| P2 | 常用地点收藏、信用勋章体系 | 影响效率与平台氛围建设 |
| P2 | 语音消息、位置消息 | 影响沟通体验，但不阻塞主流程 |

---

## 5. V2 验收方向

1. 每项需求需具备清晰的前台入口与状态反馈；
2. 关键流程需与后台治理、通知、风控形成闭环；
3. 新能力不能破坏 V1 已有主链路；
4. 上线前需同步补充对应任务执行计划、测试用例与验收文档。
