# 雀友聚 V2 需求文档

## 1. 文档目标

本文件用于承接 `my_que_you` 在 V1 之后仍未实现、但已经明确存在业务价值的需求，作为 V2 阶段的产品迭代范围管理文档。

---

## 2. V2 阶段目标

V2 目标是在 V1 主业务闭环已跑通的基础上，进一步提升平台的安全性、信任度、开团效率、消息触达与运营增长能力，并承接产品优先级文档与需求规格说明书中尚未落地的能力。

---

## 2.1 旧 SRS / 旧规划映射说明

以下条目用于说明旧《需求规格说明书》及相关旧规划中**未被 V1 吸收**的能力，当前在 V2 中的承接位置：

| 旧条目 | 当前承接位置 | 说明 |
| --- | --- | --- |
| 账号注销 | `3.1 账号与合规能力补齐 > 账号注销完整闭环` | 明确保留为 V2 范围，要求覆盖注销申请、冷静期、数据处理、账号状态流转与结果通知，不因旧 SRS 删除而丢失。 |
| 性能测试与优化 | `3.5 运营与增长能力 > 性能测试与优化` | V2 负责补齐更系统的压测、监控与专项优化，但 V1 仍保留原 SRS 的性能验收口径。 |

## 3. V2 待实现需求范围

### 3.1 账号与合规能力补齐

| 需求项           | 目标说明                                                       | 价值                               |
| ---------------- | -------------------------------------------------------------- | ---------------------------------- |
| 账号注销完整闭环 | 支持用户发起注销申请、冷静期、数据处理、账号状态流转与结果通知 | 满足合规要求，降低账号生命周期风险 |
| 实名认证         | 建立实名认证流程、认证状态展示与风控校验                       | 增强平台信任感，降低虚假身份风险   |

### 3.2 拼团效率与履约增强

| 需求项       | 目标说明                                 | 价值                           |
| ------------ | ---------------------------------------- | ------------------------------ |
| 紧急补位     | 当拼团临时缺人时，支持快速补位与优先曝光 | 提升成局率，减少临近开局流失   |
| 常用地点收藏 | 支持房主保存常用开局地点，快速复用       | 降低建局操作成本，提高发团效率 |
| 推荐排序     | 基于距离、时间、偏好等信息优化列表排序   | 提升拼团点击率与申请转化       |
| 开局提醒     | 在开局前对房主和成员进行提醒             | 提升到场率，降低爽约           |

### 3.3 社交安全与信任增强

| 需求项                   | 目标说明                                     | 价值                               |
| ------------------------ | -------------------------------------------- | ---------------------------------- |
| 黑名单屏蔽               | 支持用户拉黑、屏蔽互动与消息隔离             | 提升用户安全感与社交可控性         |
| 信用勋章系统             | 基于评价、履约、活跃等行为授予勋章标识       | 提升优质用户识别效率与平台信任氛围 |
| 标签化评价与履约档案增强 | 在基础评价外增加标签化评价、历史履约档案沉淀 | 提高匹配质量，降低纠纷             |

### 3.4 沟通体验增强

| 需求项   | 目标说明                       | 价值                       |
| -------- | ------------------------------ | -------------------------- |
| 语音消息 | 在群聊和私信中支持发送语音消息 | 提升即时沟通效率           |
| 位置消息 | 在聊天中支持发送位置消息       | 提升线下集合场景的到场效率 |
| 分享海报 | 支持拼团分享海报生成与传播     | 提升裂变传播效率           |
| 邀请码   | 支持邀请码机制与邀请归因       | 降低拉新成本               |

### 3.5 运营与增长能力

| 需求项          | 目标说明                                       | 价值                         |
| --------------- | ---------------------------------------------- | ---------------------------- |
| banner / 活动位 | 支持首页或列表活动位配置                       | 支撑城市冷启动与活动运营     |
| 消息触达增强    | 对关键状态支持更主动的消息提醒机制             | 提升用户回流与关键操作完成率 |
| 数据看板        | 建立注册、申请、成局、到场、举报等核心指标看板 | 支撑产品和运营决策           |
| 性能测试与优化  | 针对核心链路建立性能压测与优化计划             | 保证上线后稳定性与体验       |

### 3.6 社区沉淀与商业化探索

| 需求项                  | 目标说明                                 | 价值                   |
| ----------------------- | ---------------------------------------- | ---------------------- |
| 雀友圈                  | 支持用户发布内容、战绩或局后动态         | 增强社区活跃与内容沉淀 |
| 雀友俱乐部 / 固定牌友圈 | 支持建立固定社群、俱乐部内约局和成员管理 | 提升留存与复玩         |
| 长期局                  | 支持固定周期、固定成员的长期约局组织     | 承接高频稳定用户场景   |
| 战绩排行榜              | 支持战绩、胜率等榜单展示                 | 增强竞争氛围与活跃度   |
| 数字记分器 / 账单       | 提供在线记分与账单沉淀能力               | 降低线下结算纠纷       |
| 番数计算器 / 规则查询   | 支持规则查询、番数计算等辅助工具         | 降低玩法理解成本       |
| 会员权益                | 设计会员体系和权益包                     | 探索平台商业化         |
| 增值曝光                | 提供置顶、优先曝光等增值能力             | 探索供给侧商业化       |
| 多类型桌游扩展          | 将拼团能力延伸至扑克、桌游等更多场景     | 扩展用户场景与市场空间 |

## 4. V2 优先级建议

| 优先级 | 需求项                                                                                     | 原因                                   |
| ------ | ------------------------------------------------------------------------------------------ | -------------------------------------- |
| P0     | 账号注销完整闭环                                                                           | 影响合规要求与账号管理边界             |
| P1     | 实名认证、黑名单屏蔽、紧急补位、推荐排序、开局提醒                                         | 影响平台信任、安全、成局效率与履约质量 |
| P1     | 分享海报、邀请码、标签化评价与履约档案增强                                                 | 影响拉新转化与匹配质量                 |
| P2     | 常用地点收藏、信用勋章体系、语音消息、位置消息                                             | 影响效率与沟通体验，但不阻塞主流程     |
| P2     | banner / 活动位、消息触达增强、数据看板、性能测试与优化                                    | 影响运营效率和系统稳定性               |
| P2     | 雀友圈、俱乐部、长期局、战绩排行榜、记分器、番数计算器、会员权益、增值曝光、多类型桌游扩展 | 属于增长、社区和商业化延展能力         |

---

## 5. V2 验收方向

1. 每项需求需具备清晰的前台入口与状态反馈；
2. 关键流程需与后台治理、通知、风控形成闭环；
3. 新能力不能破坏 V1 已有主链路；
4. 上线前需同步补充对应任务执行计划、测试用例与验收文档；
5. 运营、增长与商业化能力需明确业务指标与灰度验证口径。
