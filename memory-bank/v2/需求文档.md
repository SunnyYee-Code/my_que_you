# 雀友聚 V2 需求文档

## 1. 文档目标

本文件用于明确 `my_que_you` 在 V1 之后、且必须在 V2 实现的需求范围，避免将中长期能力继续混入当前版本，确保版本边界、优先级和交付目标清晰。

---

## 2. V2 阶段目标

V2 目标是在 V1 主业务闭环跑通的基础上，优先补齐平台合规、安全、信任、成局效率与增长基础能力，确保平台在继续放量前具备更稳的治理能力和更高的成局效率。

---

## 2.1 版本边界说明

- **V1**：聚焦找局、建局、入局、审核、群聊、基础通知等核心闭环。
- **V2**：只承接近期必须落地的能力，包括合规、安全、履约效率和增长基础。
- **V3**：承接运营深化、社区沉淀、商业化拓展及更多体验增强型能力。

为避免范围膨胀，原先规划中不影响当前主目标的能力，不再保留在本文件内，统一移入后续计划。

## 3. V2 待实现需求范围

### 3.1 账号与合规能力补齐

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 账号注销完整闭环 | 支持用户发起注销申请、冷静期、数据处理、账号状态流转与结果通知 | 满足合规要求，降低账号生命周期风险 |
| 实名认证 | 建立实名认证流程、认证状态展示与风控校验 | 增强平台信任感，降低虚假身份风险 |

### 3.2 拼团效率与履约增强

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 紧急补位 | 当拼团临时缺人时，支持快速补位与优先曝光 | 提升成局率，减少临近开局流失 |
| 推荐排序 | 基于距离、时间、偏好等信息优化列表排序 | 提升拼团点击率与申请转化 |
| 开局提醒 | 在开局前对房主和成员进行提醒 | 提升到场率，降低爽约 |

### 3.3 社交安全与信任增强

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 黑名单屏蔽 | 支持用户拉黑、屏蔽互动与消息隔离 | 提升用户安全感与社交可控性 |
| 标签化评价与履约档案增强 | 在基础评价外增加标签化评价、历史履约档案沉淀 | 提高匹配质量，降低纠纷 |

### 3.4 增长基础能力

| 需求项 | 目标说明 | 价值 |
| --- | --- | --- |
| 分享海报 | 支持拼团分享海报生成与传播 | 提升裂变传播效率 |
| 邀请码 | 支持邀请码机制与邀请归因 | 降低拉新成本，支撑渠道归因 |

---

## 4. V2 优先级建议

| 优先级 | 需求项 | 原因 |
| --- | --- | --- |
| P0 | 账号注销完整闭环 | 影响合规要求与账号管理边界 |
| P1 | 实名认证、黑名单屏蔽 | 影响平台安全、信任与风险控制 |
| P1 | 紧急补位、推荐排序、开局提醒 | 影响成局效率、到场率与主链路体验 |
| P1 | 标签化评价与履约档案增强 | 影响匹配质量与纠纷控制 |
| P1 | 分享海报、邀请码 | 影响拉新效率与增长归因 |

---

## 5. 不纳入 V2 的需求说明

以下能力仍有业务价值，但不纳入当前 V2 范围，统一移入后续计划（V3）：

- 常用地点收藏
- 信用勋章系统
- 语音消息
- 位置消息
- banner / 活动位
- 消息触达增强
- 数据看板
- 性能测试与优化
- 雀友圈、俱乐部、长期局
- 战绩排行榜、记分器、番数计算器
- 会员权益、增值曝光
- 多类型桌游扩展

---

## 6. V2 验收方向

1. 每项需求需具备清晰的前台入口、状态反馈与异常处理提示；
2. 合规与安全能力需与后台治理、通知、风控形成闭环；
3. 新能力不能破坏 V1 已有主链路，并需明确回归范围；
4. 增长能力需具备基本归因与数据校验口径；
5. 上线前需同步补充任务执行计划、测试用例与验收文档。
