# 雀友聚 V1 验收文档

## 1. 文档目的

本文档以 `memory-bank/v1/需求文档.md` 的需求范围为基准，从交付验收视角记录：

- 哪些 V1 功能**已经验收**；
- 哪些 V1 功能**尚未验收**；
- 尚未验收功能的**待验收优先级**。

本文档不再按测试用例粒度展开，而是按 V1 需求模块做功能级验收记录。

---

## 2. 验收口径

### 2.1 已验收

以下情况可认定为“已验收”：

- 已有对应 Vitest / Playwright 自动化测试；
- 且测试已在当前仓库实际执行通过；
- 或已有明确的页面 / 交互 / 守卫行为验证证据。

### 2.2 未验收

以下情况仍认定为“未验收”：

- 功能虽已实现，但缺少直接验收证据；
- 仅有代码落点，没有实际通过的测试或验收记录；
- 强依赖真实 Supabase、真实账号、多账号联动、地图定位、对象存储、实时订阅等外部条件。

### 2.3 当前验收依据

本轮验收主要基于以下证据：

- `npm test` 结果：**21 个测试文件、74 个测试用例全部通过**
- `npx playwright test tests/e2e/v1-smoke.spec.ts` 结果：**9 个用例全部通过**
- 当前已落地的测试文件，包括：
  - 页面 / 集成测试：`src/pages/__tests__/*`
  - 权限 / 上下文 / Hook / 工具测试：`src/components/auth/__tests__/*`、`src/contexts/__tests__/*`、`src/hooks/__tests__/*`、`src/lib/__tests__/*`
  - Playwright：`tests/e2e/v1-smoke.spec.ts`
