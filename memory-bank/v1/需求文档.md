# 雀友聚 V1 需求文档

## 1. 文档目标

本文件用于沉淀 `my_que_you` Web 端在 V1 阶段的真实需求范围，并以“当前已实现能力”为基础回补需求文档，避免出现“代码已上线、文档缺失”的情况。

本次更新重点：
- 补齐当前仓库中已经实现的核心功能；
- 明确每项能力对应的页面/模块；
- 区分“已实现 / 部分实现 / 待实现”，便于后续继续排期。

---

## 2. V1 产品目标

V1 的目标是先完成同城麻将约局的最小可用闭环，让用户可以完成：
1. 注册登录；
2. 完善个人资料并进入指定城市；
3. 浏览附近拼团并筛选；
4. 发起拼团 / 申请加入拼团；
5. 房主审核申请；
6. 成员进入群聊沟通；
7. 对局结束后评价、沉淀信用；
8. 在出现纠纷时支持举报、申诉和后台处理。

## 5. V1 核心业务闭环

### 5.1 用户找局闭环

1. 用户注册/登录；
2. 首次进入完成新手引导；
3. 选择城市并授权定位；
4. 浏览附近拼团列表并筛选；
5. 查看拼团详情；
6. 提交入局申请；
7. 房主审核通过后加入拼团；
8. 进入群聊沟通并按时到场；
9. 对局结束后完成评价。

### 5.2 房主开局闭环

1. 房主创建拼团；
2. 系统校验是否允许开团；
3. 拼团展示到社区列表；
4. 房主接收申请通知；
5. 房主审核成员；
6. 成局后成员进入聊天；
7. 如有异常可取消拼团或移出成员；
8. 对局结束后沉淀评价与信用。

## 6. 当前缺口与后续建议

### 6.1 已识别但未实现的需求

- 实名认证；
- 黑名单屏蔽；
- 紧急补位；
- 常用地点收藏；
- 聊天语音消息；
- 聊天位置消息；
- 信用勋章体系；
- 更完整的账号注销闭环。

### 6.2 建议优先级

| 优先级 | 建议项 | 原因 |
| --- | --- | --- |
| P0 | 账号注销闭环补齐、状态锁定边界完善 | 影响合规与主流程稳定性 |
| P1 | 紧急补位、常用地点收藏、黑名单屏蔽 | 直接提升开团效率与使用安全感 |
| P1 | 实名认证、信用勋章体系 | 增强平台信任机制 |
| P2 | 语音消息、位置消息、更多聊天工具 | 提升体验但不阻塞主流程 |
