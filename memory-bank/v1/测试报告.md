## 3. 覆盖结论总览

### 3.1 需求覆盖结论

V1 需求文档中声明的功能模块，已全部拆解为测试场景并建立覆盖关系，覆盖模块如下：

1. 账户注册登录与新手引导
2. 城市定位与拼团发现
3. 拼团创建与房主管理

5. 群聊、私信与通知能力
6. 好友关系与邀请机制
7. 评价、信用与举报体系

### 3.2 测试类型覆盖结论

本次测试设计已覆盖以下测试类型：

- **单元测试**：覆盖输入识别、权限判断、城市缓存、列表过滤排序、时间校验、状态锁定、违禁词、评价时效、缓存失效
- **集成测试**：覆盖页面渲染、表单校验、创建/申请/审核/退出/评价/后台联动
- **端到端测试**：覆盖注册引导、找局申请、房主审核、创建拼团、邀请好友、退出扣分、互评、后台治理

## 5. 当前实际测试基线状态

### 5.1 已确认事实

通过仓库检查确认：

- 项目已配置 `vitest`，并提供 `test`、`test:unit`、`test:e2e`、`test:e2e:headed` 脚本
- 已删除示例测试：`src/test/example.test.ts`
- 当前本轮验证范围内已执行通过 **8 个 Vitest 测试文件、38 个测试用例**，覆盖登录识别、城市切换、列表排序、时间校验、详情展示、房主取消、邀请卡片、多维评价、后台治理与违禁词缓存失效等场景
- 已执行 `npx playwright test tests/e2e/v1-smoke.spec.ts`，结果为 **4 个 smoke 用例全部通过**
- 因此当前“测试覆盖”属于**测试设计覆盖完整 + 关键 Vitest 场景通过 + 可稳定运行的 Playwright smoke 通过**，但尚不是“V1 全量业务闭环自动化全部通过”

### 5.2 当前缺口

1. 尚缺大量页面级集成测试与多账号业务链路测试；
2. Playwright 浏览器依赖下载受网络影响，E2E 当前未跑通；
3. 缺少稳定测试数据、账号与环境说明；
4. 聊天、通知、好友、后台治理等复杂链路尚未形成自动化覆盖；
5. 缺少性能承接口径的量化验证证据。

---

## 6. 风险评估

| 风险项 | 影响 | 建议 |
| --- | --- | --- |
| 自动化测试覆盖几乎为空 | 无法快速回归 V1 全量功能 | 先补 P0 规则单测和主链路 E2E |
| 依赖 Supabase 实时与在线数据 | 场景复现不稳定 | 建立测试账号、测试种子数据和隔离环境 |
| 权限/状态组合复杂 | 易出现边界漏测 | 补齐角色矩阵和状态矩阵测试 |
| 治理链路涉及扣分/申诉/举报 | 一旦缺陷会直接影响用户信任 | 将治理链路纳入每次回归必测 |
| 性能目标无证据 | 无法判断是否满足旧 SRS 承接口径 | 在正式验收前补抽样性能记录 |
